# FlashPlan — Telegram-бот-ассистент с ИИ

Личный ассистент в Telegram. Пиши или говори — бот сам разберёт что это: событие в календарь, разовое напоминание, заметка или список покупок. Никаких форм и меню для ввода.

---

## Возможности

- **Распознавание из текста и голосовых сообщений** через Google Gemini API
- **Календарь событий** — добавление, просмотр и удаление по дате
- **Автонапоминания** за 7 дней, 3 дня, 1 день и 2 часа до события (настраиваются)
- **Разовые напоминания** — «напомни через 2 часа выключить плиту»
- **Заметки** — фио, номера, пароли, факты с категориями
- **Списки** — именованные чеклисты (покупки, аптека, дела)
- **Кнопка «Заметки»** — одним нажатием показать все заметки, списки и напоминания
- **Подтверждение удаления** — любое удаление требует явного подтверждения
- **Многопользовательский** — каждый видит только свои данные
- **Fallback по моделям AI** — автоматический переход к следующей модели при недоступности
- **Пул прокси** — список прокси в файле `proxies.txt`, без паролей в коде

## Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone https://github.com/ВАШ_АККАУНТ/FlashPlan.git
cd FlashPlan
```

### 2. Создать виртуальное окружение

```bash
python -m venv env
# Windows
env\Scripts\activate
# Linux / macOS
source env/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Настроить переменные окружения

```bash
cp .env.example .env
```

Открыть `.env` и вставить значения (см. раздел [Переменные окружения](#переменные-окружения)).

### 5. Настроить прокси

Создать файл `proxies.txt` в корне проекта (по одному прокси на строку):

```
# Без пароля:
153.80.113.249:63876
154.219.251.61:64202

# С паролем:
login:password@154.219.251.64:63070
```

Если прокси не нужны — оставьте файл пустым или не создавайте: бот будет работать через прямое подключение.

### 6. Запустить бота

```bash
python main.py
```

## Панель управления

В нижней части чата постоянно отображаются три кнопки:

| Кнопка | Действие |
|---|---|
| 📅 **Календарь** | Интерактивный календарь для просмотра событий по дате |
| 📝 **Заметки** | Все сохранённые заметки, списки и активные напоминания |
| ⚙️ **Настройки** | Управление автонапоминаниями к событиям |

Все окна закрываются только вручную кнопкой **✕ Закрыть**.

## Как добавить событие

Напишите боту в свободной форме:

```
Встреча с командой в пятницу в 15:30
Врач 12 мая в 10 утра
Созвон с клиентом завтра в 9
```

Или отправьте **голосовое сообщение** — бот распознает его точно так же.

Бот покажет что понял и попросит подтверждение:

```
Я понял так:
📅 12.05.2026 в 10:00
📝 Врач

Всё верно?  [✅ Согласен]  [🎤 Перезапись]
```

## Second Brain — заметки, списки, напоминания

Бот автоматически определяет тип сообщения:

| Пример | Тип | Действие |
|---|---|---|
| «Напомни через 2 часа выключить плиту» | ⏰ Напоминание | Одноразовое уведомление в нужное время |
| «Фио доктора Иванов Иван Петрович» | 📝 Заметка | Сохранит под категорией «доктор» |
| «Что за фио у доктора?» | 🔍 Запрос | Покажет заметку с кнопкой Закрыть |
| «Купить молоко, хлеб, масло» | 🛒 Список | Добавит в список «покупки» |
| «Добавь колбасу в покупки» | 🛒 Список | Дополнит список |
| «Удали список покупки» | 🗑 Удаление | Попросит подтверждение |
| «Забудь про доктора» | 🗑 Удаление | Попросит подтверждение |

### Кнопка «📝 Заметки»

Показывает одним сообщением все данные пользователя:
- Все сохранённые заметки с категориями
- Все именованные списки с элементами
- Все активные напоминания с временем

```
Удалить список «покупки» целиком?
[🗑 Удалить]  [❌ Отмена]
```

## Автонапоминания к событиям

Бот автоматически присылает уведомления по мере приближения события:

| Порог | Сообщение |
|---|---|
| 7 дней | «Осталась неделя» |
| 3 дня | «Осталось 3 дня» |
| 1 день | «Завтра!» |
| 2 часа | «Осталось 2 часа!» |

### Настройка

Нажмите **⚙️ Настройки**. Откроются переключатели для каждого порога:

```
⏰ 2 ч   ✅    −  +
📅 1 д   ✅    −  +
📅 3 д   —     −  +
📅 7 д   ✅🔒  −  +
```

Нажатие на значение включает/выключает порог. Кнопки − / + меняют значение (мин. 1, макс. 24 для часов / 30 для дней). Минимум один порог должен быть активен (🔒).

## Как работает AI-распознавание

1. Текст или аудио отправляется в Google Gemini API
2. Бот перебирает модели по приоритету (от лёгких к тяжёлым):
   - `gemini-3.1-flash-lite-preview`
   - `gemini-3-flash-preview`
   - `gemini-2.5-flash-lite`
   - `gemini-2.5-flash`
3. При ошибке одной модели автоматически переходит к следующей
4. Случайно выбирается один из двух API-ключей и один из прокси в `proxies.txt`
5. Gemini возвращает JSON с intent и параметрами
6. Если все модели недоступны — бот сообщает об ошибке

## База данных

SQLite, файл `events.db` создаётся автоматически рядом с `main.py`.

**Таблица `events`** — события календаря:

| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER | Первичный ключ |
| `user_id` | INTEGER | Telegram ID пользователя |
| `event_text` | TEXT | Описание события |
| `event_time` | TIMESTAMP | Дата и время события |
| `rem_7d/3d/1d/2h` | BOOLEAN | Отправлено ли напоминание |
| `person` | TEXT | Имя владельца события (опционально) |

**Таблица `short_reminders`** — разовые напоминания:

| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER | Первичный ключ |
| `user_id` | INTEGER | Telegram ID |
| `text` | TEXT | Текст напоминания |
| `trigger_at` | TIMESTAMP | Время срабатывания |

**Таблица `knowledge_base`** — заметки:

| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER | Первичный ключ |
| `user_id` | INTEGER | Telegram ID |
| `content` | TEXT | Содержимое заметки |
| `category` | TEXT | Категория (ключевое слово) |
| `created_at` | TIMESTAMP | Дата создания |

**Таблица `checklists`** — элементы списков:

| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER | Первичный ключ |
| `user_id` | INTEGER | Telegram ID |
| `list_name` | TEXT | Название списка |
| `item_text` | TEXT | Текст элемента |
| `created_at` | TIMESTAMP | Дата добавления |
